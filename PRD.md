## 1. 项目概述

### 1.1 产品定位

ED Assistant 是一款面向个人和小团队的跨平台桌面文件加解密工具。以 [age](https://github.com/FiloSottile/age) 作为核心加密引擎，通过简洁、现代的图形界面，让非技术用户也能安全、便捷地对文件进行加密保护与解密还原。

### 1.2 核心价值

- **安全**：采用 age 格式（X25519 + ChaCha20-Poly1305），密码学设计经过严格审查，无配置项可供误用
- **易用**：图形化操作取代命令行，拖拽文件即可完成加解密
- **灵活**：同时支持密码模式与密钥模式，支持多收件人加密
- **跨平台**：macOS、Windows、Linux 三端一致体验

### 1.3 加密引擎

集成 [filippo.io/age](https://github.com/FiloSottile/age) v1.3.1+，通过以下方式调用：

- 将 age / age-keygen 的预编译二进制文件打包进应用（通过 `electron-builder` 的 `extraResources`）
- 在 Electron 主进程中通过 Node.js `child_process` 调用，不暴露任何 age 能力给渲染进程（安全边界）
- 支持的 age 特性：passphrase、X25519 公钥、SSH 公钥、后量子密钥（`-pq`）、armor 模式、多收件人

## 3. 核心功能需求

### 3.1 功能总览

| ID | 功能模块 | 优先级 |
|----|---------|-------|
| F-00 | 密码生成器 | P0 |
| F-01 | 密码加密 | P0 |
| F-02 | 密码解密 | P0 |
| F-03 | X25519 密钥对生成 | P0 |
| F-04 | 公钥加密（单收件人） | P0 |
| F-05 | 私钥解密 | P0 |
| F-06 | 多收件人加密 | P1 |
| F-07 | SSH 公钥加密 | P1 |
| F-08 | SSH 私钥解密 | P1 |
| F-09 | Armor 模式（PEM 文本输出） | P1 |
| F-10 | 密码保护的私钥文件 | P1 |

| F-14 | 收件人列表文件（.txt）管理 | P2 |

### 3.2 F-00 密码生成器

密码生成器是独立于加解密流程的辅助工具，帮助用户生成高强度随机密码，可直接用于文件加密口令或其他场景。

#### 3.2.1 生成配置项

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| 密码长度 | 数字滑块 + 输入框 | 20 | 范围 8 ～ 128 |
| 大写字母（A-Z） | 开关 | 开启 | |
| 小写字母（a-z） | 开关 | 开启 | |
| 数字（0-9） | 开关 | 开启 | |
| 特殊字符 | 开关 + 自定义输入 | 开启，默认字符集 `!@#$%^&*()-_=+[]{}` | 用户可自定义追加或替换字符集 |
| 排除易混淆字符 | 开关 | 关闭 | 排除 `0`、`O`、`l`、`1`、`I` 等视觉相近字符 |
| 排除重复字符 | 开关 | 关闭 | 生成的密码中每个字符仅出现一次（长度受字符集大小限制） |

> 至少需启用一类字符，否则"生成"按钮不可点击，并给出提示。

#### 3.2.2 生成结果展示

- 大字号展示生成的密码，默认以 `●●●●` 遮罩，提供"显示 / 隐藏"切换按钮
- **密码强度评级**：根据长度与字符集复杂度实时评级，显示为 弱 / 中 / 强 / 极强，并附熵值估算（bits）
- **一键复制**：点击后密码写入剪贴板，按钮短暂变为"已复制 ✓"状态（2 秒后还原）
- **刷新生成**：点击"重新生成"按钮或快捷键（⌘R / Ctrl+R）立即用相同配置生成新密码
- **历史记录**：本次会话内保留最近 5 条生成记录（仅内存，关闭应用后清除），供用户对比选择，列表中每条记录均可单独复制或删除

#### 3.2.3 与加密页面的联动

- 加密页面"密码模式"的口令输入框旁提供"从密码生成器填入"按钮（🎲 图标）
- 点击后弹出迷你密码生成器浮层，生成满意后点击"使用此密码"将其自动填入加密口令框

### 3.3 F-01 密码加密

- 用户通过文件选择器或拖拽选择一个或多个输入文件
- 用户在密码输入框中输入加密口令（支持显示/隐藏切换）
- 提供"强度指示"组件，实时提示口令强度（弱 / 中 / 强）
- 支持让 age 自动生成安全口令，并在界面上显示可复制的结果
- 输出文件默认追加 `.age` 后缀，用户可自定义输出路径
- 支持 Armor 模式，输出 PEM 格式文本文件（`.age` 或自定义后缀）
- 完成后显示操作摘要（文件名、大小、耗时）

**对应 age 命令：**
```
age --passphrase [--armor] -o <output> <input>
```

### 3.4 F-02 密码解密

- 用户选择 `.age` 文件（或其他 age 格式文件）
- 输入口令后执行解密，输出至用户指定目录
- age 自动识别 passphrase 加密格式，无需用户手动选择模式
- 解密失败（口令错误）时展示明确的错误提示，不暴露底层错误栈

**对应 age 命令：**
```
age --decrypt -o <output> <input>
```

### 3.5 F-03 X25519 密钥对生成

- 界面提供"生成新密钥对"入口
- 生成后展示：
  - 公钥（`age1...`），可一键复制
  - 私钥（`AGE-SECRET-KEY-1...`），默认隐藏，需用户主动点击"显示"
- 提供"保存私钥文件"按钮，通过系统文件保存对话框存储到用户选定位置
- 提示用户妥善保管私钥，丢失后无法恢复

**对应 age 命令：**
```
age-keygen -o key.txt
```

### 3.6 F-04 / F-05 公钥加密与私钥解密

- **加密**：用户输入或粘贴一个公钥（`age1...`），或通过文件选择器加载公钥文件
- **解密**：用户选择对应私钥文件（`key.txt`）
- 若私钥文件本身经密码保护，弹出二次口令输入框

### 3.7 F-06 多收件人加密

- 支持在收件人列表中添加多个公钥（`age1...` 或 `ssh-ed25519 AAAA...`、`ssh-rsa AAAA...`）
- 每个收件人可单独删除，并显示人类可读的备注名称（用户自填）
- 支持导入收件人列表文件（每行一个公钥，`#` 开头为注释）
- 支持导出当前收件人列表为 `.txt` 文件

### 3.8 F-07 / F-08 SSH 密钥支持

- **加密**：接受 `ssh-ed25519 AAAA...` 或 `ssh-rsa AAAA...` 格式的 SSH 公钥
- **解密**：通过文件选择器选择 SSH 私钥文件（`id_ed25519`、`id_rsa` 等）
- 提示用户 SSH 密钥加密文件会在 header 中嵌入公钥指纹，文件可被关联追踪（隐私提示）

### 3.9 F-09 Armor 模式

- 全局或每次操作时勾选"Armor 输出"开关
- 启用后输出标准 PEM 格式，适合嵌入邮件/文本

### 3.10 F-10 密码保护的私钥文件

- 在密钥对生成界面，提供"为私钥文件加密"选项
- 启用后执行 `age-keygen | age -p > key.age`，生成口令保护的私钥
- 解密时若检测到 `.age` 格式私钥文件，自动弹出二次口令输入

### 3.11 F-11 后量子密钥（P2）

- 在密钥生成界面提供"后量子模式"开关（标注：实验性功能）
- 生成 `AGE-SECRET-KEY-PQ-1...` / `age1pq1...` 格式密钥对
- 界面提示后量子公钥长度约 2000 字符的注意事项

### 3.12 F-12 加密文件元数据检视（P2）

- 提供"检视文件"功能，不解密仅展示 age 文件元数据
- 显示：加密接收方类型、是否后量子、header/payload 大小
- 对应 `age-inspect [--json]`

### 3.13 F-13 批量加解密（P2）

- 支持同时选择多个文件（或整个文件夹），使用相同参数批量处理
- 展示批量进度列表，每个文件独立显示状态（等待 / 处理中 / 完成 / 失败）
- 支持遇错继续（跳过失败文件，汇总错误报告）

## 4. 加密方案详述

### 4.1 支持的加密接收方类型

| 类型 | 接收方格式 | 对应私钥 |
|------|-----------|---------|
| X25519 | `age1...`（Bech32） | `AGE-SECRET-KEY-1...` |
| SSH Ed25519 | `ssh-ed25519 AAAA...` | `~/.ssh/id_ed25519` 等 |
| SSH RSA | `ssh-rsa AAAA...` | `~/.ssh/id_rsa` 等 |
| Passphrase | 无公钥，口令派生 | 相同口令 |
| Post-quantum | `age1pq1...`（~2000 字符） | `AGE-SECRET-KEY-PQ-1...` |

### 4.2 age 二进制集成方案

```
应用包/
└── resources/
    └── bin/
        ├── age-darwin-arm64      # macOS Apple Silicon
        ├── age-darwin-amd64      # macOS Intel
        ├── age-win32-x64.exe     # Windows
        ├── age-linux-x64         # Linux
        ├── age-keygen-darwin-arm64
        └── ...（age-keygen 各平台同理）
```

- 主进程启动时根据 `process.platform` + `process.arch` 选择对应二进制
- 调用时通过 `child_process.spawn`，输入输出通过 stdin/stdout pipe 传递，**不写临时文件**（防止敏感数据落盘）
- 对于大文件（> 100 MB）使用 Node.js Stream 流式处理，避免全量内存占用

### 4.3 主进程 IPC 接口设计（概要）

```typescript
// 渲染进程 → 主进程（通过 preload 暴露）
interface AgeApi {
  encrypt(params: EncryptParams): Promise<EncryptResult>
  decrypt(params: DecryptParams): Promise<DecryptResult>
  generateKeypair(pq?: boolean): Promise<Keypair>
  inspect(filePath: string): Promise<AgeFileInfo>
}

interface EncryptParams {
  inputPaths: string[]          // 待加密文件
  outputDir: string             // 输出目录
  mode: 'passphrase' | 'pubkey'
  passphrase?: string           // mode=passphrase 时使用
  recipients?: string[]         // mode=pubkey 时使用，支持多个
  armor?: boolean
}

interface DecryptParams {
  inputPath: string
  outputDir: string
  mode: 'passphrase' | 'identity'
  passphrase?: string
  identityPath?: string
  identityPassphrase?: string   // 私钥文件本身有口令保护时
}
```

## 6. 密钥管理

### 6.1 密钥存储原则

- **不在应用内存储任何私钥**，私钥由用户自行保管在文件系统
- 收件人公钥列表（仅公钥）可选择持久化至 `electron-store` 加密存储区
- 口令在内存中使用后立即清零（`Buffer.fill(0)`），不写入日志

### 6.2 密钥导入途径

| 途径 | 支持格式 |
|------|---------|
| 粘贴文本 | `age1...`、`ssh-ed25519 ...`、`ssh-rsa ...`、`AGE-SECRET-KEY-1...` |
| 文件选择 | `key.txt`、`key.age`、`id_ed25519`、`id_rsa`、`*.pub` |
| 拖拽文件 | 同上 |

### 6.3 密钥验证

- 输入公钥时实时校验格式合法性，非法格式给出具体错误描述
- 私钥文件选择后检查文件头格式，提前提示"文件格式不匹配"

### 7.3 安全边界

```
渲染进程（React）
    │  不可直接访问 Node.js / 文件系统
    │  只能调用 window.electronAPI 暴露的有限接口
    ↓

    │  白名单式暴露 IPC 调用

```

## 8. 非功能性需求

### 8.1 性能

| 指标 | 目标 |
|------|------|
| 应用冷启动时间 | < 3 秒 |
| 100 MB 文件加密耗时 | < 5 秒（Apple M 系列 / 同等 x86） |
| 1 GB 文件加密内存峰值 | < 200 MB（流式处理） |
| UI 操作响应 | < 100 ms（加解密异步，不阻塞 UI） |

### 8.2 安全

- 口令不得出现在进程命令行参数中（防止 `ps aux` 泄漏），通过 stdin pipe 传入 age
- 临时文件：禁止将明文或口令写入磁盘临时文件
- 日志：应用日志不记录口令、私钥内容
- 权限：应用不请求超出文件读写所需的系统权限

### 8.3 兼容性

## 10. 术语表

| 术语 | 说明 |
|------|------|
| age | [filippo.io/age](https://github.com/FiloSottile/age)，现代文件加密工具 |
| Recipient（收件人） | 能解密文件的公钥持有者 |
| Identity（身份/私钥） | 用于解密的私钥文件 |
| Passphrase（口令） | 密码模式下的加密密码 |
| Armor | age 的 PEM 文本输出格式，适合嵌入文本 |
| X25519 | age 原生公钥加密算法（Diffie-Hellman over Curve25519） |
| Post-quantum | age v1.3.0+ 支持的 ML-KEM-768+X25519 混合后量子密钥 |
| IPC | Electron 进程间通信（Inter-Process Communication） |
| contextBridge | Electron preload 中安全暴露主进程能力的 API |
